# 26. 右上角可配置信息面板说明文档

## 1. 本次改动的目标

这次已经将游戏界面右上角原本“写死内容”的说明区域，整理为一个**可由剧本单独配置的信息面板**。

目前它仍然会继续显示你之前看到的内容，例如：

- 当前剧本：默认主线剧本 / 游戏测试剧本
- 事件总数：若干
- 测试剧本中的“上手提示”

也就是说，这次改动的重点是：

- **显示效果先保持原样**
- **底层改成可配置结构**
- 以后你如果想改显示文案，优先改 YAML，不需要再改 C# 代码

---

## 2. 现在是由哪里控制这块面板的

### 2.1 面板代码文件

右上角信息面板的代码在：

- `Scripts/UI/ConfigurableInfoPanel.cs`

它负责：

- 读取当前剧本对应的 `info_panel.yaml`
- 根据当前状态选择要显示哪一段文字
- 替换占位符，例如剧本名、事件数

### 2.2 剧本资源中的配置入口

每个剧本资源里都新增了一个字段：

- `InfoPanelConfigPath`

定义位置在：

- `Scripts/Core/Scenario/GameScenarioDefinition.cs`

目前两个剧本分别指向：

- 主线剧本：`Configs/Scenarios/MainStory/UI/info_panel.yaml`
- 测试剧本：`Configs/Scenarios/TestGame/UI/info_panel.yaml`

---

## 3. 你之后主要改哪里

如果你只是想改右上角显示的文字内容，优先改下面两个文件：

- 主线剧本右上角信息面板配置  
  `Configs/Scenarios/MainStory/UI/info_panel.yaml`

- 测试剧本右上角信息面板配置  
  `Configs/Scenarios/TestGame/UI/info_panel.yaml`

你一般**不需要改代码**，除非你想新增新的显示规则、特殊状态判断或新的占位符。

---

## 4. 目前支持的配置结构

当前 `info_panel.yaml` 使用的是这种结构：

```yaml
summary_lines:
  - "当前剧本：{{scenario_display_name}}"
  - "事件总数：{{event_count}}"

states:
  no_scenario:
    content_lines:
      - "请先从主菜单进入一个剧本。"

  test_scenario:
    content_lines:
      - "上手提示："
      - "1. 先点“捡起石斧”"
      - "2. 再点“点击砍树”或“挂机砍树”"
      - "3. 技能有经验后再去“技能页手动升级”"

  story_empty:
    content_lines:
      - "当前主线剧本还没有配置事件。"
      - "这正是主线与测试剧本分离后的正常状态。"

  story_default:
    content_lines:
      - "当前为正式剧本模式。"
      - "建议在“系统”标签里手动保存存档。"
```

---

## 5. 当前已支持的状态分支

右上角信息面板目前会根据游戏状态，自动选择对应内容：

- `no_scenario`
  - 还没有进入任何剧本时显示

- `test_scenario`
  - 当前进入的是测试剧本时显示

- `story_empty`
  - 当前是正式主线剧本，但还没有配置任何事件时显示

- `story_default`
  - 当前是正式主线剧本，且已有正常内容时显示

---

## 6. 当前可直接使用的占位符

你可以在 YAML 里直接写这些占位符：

- `{{scenario_display_name}}`
  - 当前剧本显示名

- `{{event_count}}`
  - 当前已注册事件总数

- `{{scenario_id}}`
  - 当前剧本内部 ID

- `{{scenario_description}}`
  - 当前剧本描述

- `{{is_test_scenario}}`
  - 是否为测试剧本，结果为 `true` 或 `false`

- `{{system_tab_enabled}}`
  - 当前剧本是否启用了系统标签，结果为 `true` 或 `false`

## 9. 你以后最常用的修改方式

如果你后续想修改右上角内容，推荐按下面顺序操作：

1. 先确定你要改的是主线剧本还是测试剧本
2. 打开对应剧本目录下的 `UI/info_panel.yaml`
3. 直接修改 `summary_lines` 或对应 `states` 下的 `content_lines`
4. 重新运行游戏查看效果

如果以后你想要：

- 新增更多状态分支
- 新增更多占位符
- 让右上角显示更复杂的动态信息

那时再改：

- `Scripts/UI/ConfigurableInfoPanel.cs`

即可。
