# 61_多版本UI设计与左侧边栏主题规则说明_0415

## 1. 文档目的
- 记录今天新增/修正的多版本 UI（legacy/stitch）设计规则。
- 重点固化左侧边栏（区域/技能）在亮色新UI下的按钮态、技能层级字号、技能分组配色映射规范。
- 作为后续扩展“白天亮色 / 夜晚深色 / 其他主题”时的统一参考。

## 2. 当前多版本UI架构（已落地）
- 主题模式来源：`PlayerUiState.UiThemeMode`
- 主题切换入口：系统标签页（`SystemPanel.Configure(..., UseStitchUiTheme, UseLegacyUiTheme)`）
- 运行时切换主流程：
  1. `SwitchUiThemeMode(mode)`
  2. `LoadLayoutSettings()` + `GameplayUiConfigLoader.LoadTheme(...)`
  3. `RebuildUiStructure()`
  4. `RefreshAllPanels()`
- 图片主题联动：`UiImageThemeManager.SetThemeMode(...)`
- 结论：当前已具备“同一套业务逻辑 + 多套UI表现层”的切换基础。

## 3. 左侧边栏本次修正（新UI stitch）

### 3.1 顶部“区域/技能”双按钮态统一
- 问题：未激活态在亮色主题下近似纯文字，按钮轮廓不明显。
- 处理：`ApplyLeftSidebarToggleVisual(Button button, bool isActive)` 增加 stitch 分支，激活/未激活都使用完整按钮框（背景+描边+悬停+按下态）。
- 结果：
  - `区域` 和 `技能` 始终都为按钮外观。
  - 激活态与未激活态可稳定区分。

### 3.2 技能模式字号对齐区域模式
- 目标：一级技能/二级技能字号与区域模式一级/二级层级保持一致。
- 新增方法：
  - `ResolveSkillGroupFontSize()`：对齐一级层级字号。
  - `ResolveSkillNodeFontSize()`：对齐二级层级字号。
- 应用位置：
  - `BuildSkillGroupNode(...)`
  - `BuildSkillNode(...)`

### 3.3 技能分组颜色分层（亮色主题）
- 目标：避免亮黄等高亮色在浅背景刺眼；建立“同组同系，不同组有区分”的深色体系。
- 新增色板：
  - `StitchSkillGroupPrimaryPalette`（一级技能深色）
  - `StitchSkillGroupSecondaryPalette`（二级技能较深色）
- 新增映射逻辑：
  - `GetSkillGroupColors(groupId)`
  - `ResolveSkillGroupPaletteIndex(groupId)`（稳定哈希映射）
- 规则：
  - 同一 `groupId` 的一级/二级技能固定落在同一色系（深色 + 同系次深色）。
  - 不同 `groupId` 通过色板索引得到不同深色系。

## 4. 新增通用样式能力
- 新增 `CreateSidebarButtonStyle(...)`：为左栏按钮提供统一的背景、描边、阴影参数，降低后续主题扩展成本。
- 现已用于：
  - 左侧顶部“区域/技能”切换按钮（stitch）
  - 技能树一级/二级节点按钮（stitch）

## 5. 兼容性与影响面
- 旧版 UI（legacy）分支保持原样，不改变既有视觉规则。
- 本次仅强化 stitch 分支视觉，不改业务逻辑。
- 技能选择、区域切换、数据刷新流程不变。

## 6. 后续新增主题的实施建议（强约束）
- 新主题必须提供：
  1. 顶部双切换按钮完整状态（normal/hover/pressed/active）
  2. 左栏一级/二级层级字号映射（与区域模式层级对齐）
  3. 组色映射规则（同组同系，不同组可区分）
  4. 文字/背景对比度校验（避免浅底浅字、深底深字）
- 优先复用：
  - `ApplyLeftSidebarToggleVisual(...)` 的主题分支模式
  - `CreateSidebarButtonStyle(...)`
  - `GetSkillGroupColors(...)` + 色板策略
- 如要新增“日间亮色/夜间深色”之外的第三套主题：
  - 不改业务层；仅新增主题资源 + 视觉分支。
  - 保持切换接口不变，继续走 `SwitchUiThemeMode(...)` 全量重建流程。
