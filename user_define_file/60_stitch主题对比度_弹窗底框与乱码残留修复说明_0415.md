# 60. stitch主题对比度、弹窗底框与乱码残留修复说明（0415）

## 一、目标与范围
本次修改对应以下需求：
1. 排查并修复其他文件中的疑似乱码残留，并在问题位置加备注。
2. 新UI（stitch）下将“当前区域 > 子场景”底色从偏白改为更压低的背景色。
3. 修复底部日志“最小化/最大化/展开”按钮文字对比度，并检查亮色主题下其他低对比文本（重点含右上角信息面板）。
4. 使用 `stitch_Element_UI` 设计语义为“一次性事件弹窗/普通弹窗”提供统一底框。
5. 使用更深色底框作为“日志全屏弹窗”与“按钮悬浮提示（Tooltip）”通用底框，确保文字可读性。

## 二、乱码残留排查与修复

### 1) 排查方式
- 对 `Scripts` 下 `.cs` 文件进行乱码特征关键字扫描（如 `锟`、`褰`、`鐩`、`璇` 等）。
- 对项目内 `.cs/.yaml/.md/.txt` 做二次扫描确认无新增残留。

- `"当前区域"`（原乱码）
- `$"{skillName}相关"`（原乱码）
- `$"★{scene.Title}"`（原乱码）

### 3) 备注标记

## 三、子场景底色（stitch）下压
为避免“子场景卡片白底过亮影响按钮/文字焦点”，将 stitch 主题下 scene card 底色调整为更低亮度层级：
- 文件：`Configs/UI/gameplay_ui_theme_stitch.yaml`
- 调整：`panels.scene_card.color`
- 从：`#ffffff`
- 到：`#eeeee8`

该调整仅影响 stitch 主题，不影响 legacy 主题。

## 五、弹窗底框与深色通用底框

### 1) 新增样式库
- 新文件：`Scripts/UI/StitchElementStyleLibrary.cs`
- 提供：
  - `CreateLightDialogFrame(...)`：亮色弹窗底框（一次性事件/普通确认/存档弹窗）
  - `CreateLightSubPanelFrame(...)`：亮色内层文本框（存档项）
  - `CreateDeepOpaqueFrame(...)`：深色不透明底框（日志全屏）
  - `CreateDeepTooltipFrame(...)`：深色 Tooltip 底框

### 2) 一次性事件弹窗 / 普通弹窗
- 文件：
  - `Scripts/UI/EventDialogPanel.cs`
  - `Scripts/UI/ConfirmActionDialog.cs`
  - `Scripts/UI/SaveSlotDialog.cs`
- `Configure` 均新增主题参数：`useStitchStyle`。
- stitch 下应用亮色弹窗底框和对应按钮文字配色；legacy 不变。

### 3) 日志全屏弹窗与 Tooltip 通用深色底框
- 文件：`Scripts/UI/GamePlayUI.cs`
  - 日志最大化弹窗容器改用 `CreateDeepOpaqueFrame(...)`（不透明深色底框）。
- 文件：`Scripts/UI/GamePlayUI.Actions.cs`
  - 新增 `ApplyRuntimeTooltipTheme()`，在 stitch 模式下统一设置 `TooltipPanel` 深色底框与高对比文字。
- 说明：该 Tooltip 样式作为该主题下按钮悬浮弹出信息的通用底框逻辑。
