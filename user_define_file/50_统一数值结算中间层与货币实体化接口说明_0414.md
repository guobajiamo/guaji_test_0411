# 50_统一数值结算中间层与货币实体化接口说明_0414

## 1. 目标与结论

本次已将关键数值流程统一收口到“中间层结算接口”，不再由业务系统直接即时改写库存/货币/经验。

当前统一入口为：

- `Scripts/Core/Helpers/ValueSettlementService.cs`

倍率接口与限时 Buff 入口为：

- `Scripts/Systems/BuffSystem.cs`
- `Scripts/Core/Helpers/SettlementStatIds.cs`

## 2. 已接入中间层的业务流程

以下流程已统一通过 `ValueSettlementService` 结算：

1. 挂机产出总量结算（道具入包）
2. 挂机进度时长缩减（速度倍率）
3. 挂机经验结算（经验倍率）
4. 商店买入支付（金币成本+折扣倍率）
5. 商店卖出收益（卖价倍率）
6. 点击事件成本与奖励（道具/金币/经验）
7. 任务领奖效果（道具/金币/经验）
8. 一键卖垃圾（道具扣除+金币发放）

## 3. 非即时结算说明

现在业务系统（`IdleSystem/ShopSystem/SellSystem/ClickEventSystem/QuestSystem`）只负责“提出结算请求”，
实际数值变更统一由中间层执行：

- 道具增减：`AddItem` / `TryRemoveItem` / `TryPayItemCosts`
- 货币增减：`AddCurrency` / `TrySpendCurrency`
- 经验发放：`GrantSkillExp`
- 价格与效率计算：`Resolve*` 系列方法

这意味着后续加入复杂系统（Buff、区域规则、剧情系数、活动倍率）时，只需要在中间层/倍率层扩展，无需到每个业务系统重复改逻辑。

## 4. 倍率与 Buff 接口（可扩展）

`BuffSystem` 支持：

- 常驻倍率：`SetPersistentMultiplier`
- 限时倍率：`SetTimedMultiplier`
- 清理来源：`RemoveBuff` / `RemoveMultiplier`
- 读取倍率：`GetMultiplier`

当前已接入的倍率键（见 `SettlementStatIds`）：

- `idle.output.multiplier`（挂机收获加成）
- `idle.speed.multiplier`（挂机进度提速）
- `trade.buy_price.multiplier`（买入折扣/加价）
- `trade.sell_price.multiplier`（卖出收益系数）
- `skill.exp_gain.multiplier`（经验加成）

并预留了按技能/NPC/物品维度的细分键：

- `idle.output.{skillId}.multiplier`
- `idle.speed.{skillId}.multiplier`
- `skill.exp_gain.{skillId}.multiplier`
- `trade.buy_price.{npcId}.multiplier`
- `trade.sell_price.{itemId}.multiplier`

## 5. 货币实体化（与中间层联动）

货币系统现在是“双表示”：

1. 经济账本（原有）：`PlayerEconomyState`
2. 背包实体（新增同步）：`currency_*` 物品堆叠

同步策略：

- 所有货币增减都通过 `ValueSettlementService`
- 中间层在增减账本时同步增减背包货币实体
- 读档后会执行一次账本 -> 背包同步，确保旧档兼容

默认金币实体 ID：

- `currency_gold`

## 6. 技能升级特殊事件接口（保持可用）

技能升级触发特殊事件接口保持可用且未破坏：

- `Skills/Definitions/*` 中每级可配置 `on_level_up_event_ids`
- 运行时由 `SkillSystem` 调用 `ClickEventSystem.TryTriggerEventFromSystem(...)`

因此例如“5级触发 buff 解锁”“10级触发一次性支线事件”可继续按 YAML 配置，不需要改业务代码。
