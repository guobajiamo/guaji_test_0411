# 49_左侧选中区域角标改为沿边同弧度绘制说明_0414

## 1. 本次调整原因

用户继续反馈左侧“当前所在二级区域”的橙色括号仍然不够贴边。

核心问题并不只是“位置往外挪得不够”，而是上一版实现方式本身有先天限制：

- 上一版是 4 个独立的小圆角边框块
- 这些小块即使挪到更外侧，圆角弧度也不一定和蓝色选中底块一致
- 所以会出现“看起来靠近了，但线条还是贴不住边缘”的问题

也就是说，这次问题的关键不是单纯继续调 `inset`，而是要更换绘制方式。

---

## 2. 本次采用的新方案

这次不再使用“独立小圆角边框块”来模拟四角括号，而是改成：

- 直接按蓝色选中背景的圆角轮廓绘制四角括号

这样做的好处是：

1. 括号和蓝色底块使用同一套圆角几何
2. 线条可以真正压在底块边缘上，而不是看起来像浮在里面
3. 后续如果整体 UI 做自适应缩放，只要主题圆角变化，括号也会跟着同步调整

---

## 3. 实际修改内容

涉及文件：

- `Scripts/UI/GamePlayUI.Actions.cs`
- `Scripts/UI/GamePlayUI.Layout.cs`

### 3.1 新增自绘制角标控件

在 `Scripts/UI/GamePlayUI.Actions.cs` 中新增：

- `CornerBracketOverlayControl`

它不再依赖 `StyleBoxFlat` 的边框模拟，而是通过绘制：

- 四段圆角弧线
- 四组短横线
- 四组短竖线

来组成四角括号。

### 3.2 角标与选中底块共用圆角逻辑

新的角标控件会使用：

- `_theme.CornerRadius`

作为基础圆角值。

这意味着左侧二级区域选中态的蓝色底块和橙色角标，在几何上会更一致。

### 3.3 角标外扩显示

新的覆盖层不再被限制在蓝色面板内部，而是会向外额外扩出一圈绘制空间。

作用：

- 让括号线条真正压住选中底块的边缘
- 避免橙色线条又被挤回蓝色底块内部

### 3.4 改为主题联动参数

在 `BuildSecondaryAreaNode(...)` 中，角标参数不再写死，而是改为根据 `_theme.CornerRadius` 推导：

- 括号长度
- 向外压边距离
- 线条粗细

好处：

- 当前版本更贴边
- 后续如果整体 UI 做自适应缩放，这组参数也更容易保持同步比例

---

## 4. 为什么这次方案更适合

如果仍沿用旧方案，只继续把括号往外挪，会遇到两个问题：

1. 位置可以更外，但弧度还是和蓝色底块不一致
2. 不同分辨率或后续缩放下，写死的小圆角块会更容易变形

所以本次更合理的做法不是继续“硬挪位置”，而是直接把角标改成沿同一圆角边缘绘制。

这也是本次最重要的优化点。

---

## 5. 后续若还要继续微调，改哪里

主要入口：

- `Scripts/UI/GamePlayUI.Actions.cs`
  `CornerBracketOverlayControl`

可调项：

- `_cornerRadius`
  控制括号弧度，当前会跟随主题圆角
- `_armLength`
  控制括号两端延伸长度
- `_thickness`
  控制括号线条粗细
- `_panelOutset`
  控制绘制层向外扩出的空间

调用入口：

- `Scripts/UI/GamePlayUI.Layout.cs`
  `BuildSecondaryAreaNode(...)`

这里控制根据主题参数推导角标尺寸的方式。
