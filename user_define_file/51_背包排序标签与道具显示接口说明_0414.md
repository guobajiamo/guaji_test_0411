# 51_背包排序标签与道具显示接口说明_0414

## 1. 本次更新目标

本次针对“背包重做前的接口准备”完成了三件事：

1. 补齐背包顺序逻辑接口：入袋顺序、自动排序、删除后前补位。  
2. 补齐背包标签筛选接口：按标签只显示对应道具。  
3. 补齐道具信息显示接口：支持每个物品自定义悬浮字段与详细页字段。  

---

## 2. 背包排序逻辑（已实装）

### 2.1 默认顺序（不点自动排序）

- 物品首次入袋时，记录 `AcquiredSequence`（入袋序号）与 `PlayerDisplayOrder`（显示序号）。
- 物品在背包中按 `PlayerDisplayOrder` 显示，默认就是“先来先占位”。
- 当中间某格物品被消耗到 0 时，会自动把后续序号整体前移（补位）。
- 新入袋物品会拿到当前末尾序号，放到最后。

### 2.2 自动排序按钮

- 点击“自动排序”后，按“分类类型路径 + 物品定义顺序 + 文件顺序 + 条目顺序 + ID”重排当前背包显示序号。
- 这是一次重排行为，重排结果会写入 `PlayerDisplayOrder`，后续增删继续基于当前序列补位/追加。

### 2.3 恢复入袋顺序按钮

- 点击“入袋顺序”可按 `AcquiredSequence` 重排，恢复为“先来先占位”的顺序。

## 3. 背包标签筛选（已实装）

- 新增标签筛选按钮（全部、消耗品、材料、工具、装备、武器、食物、燃料、任务、货币、斧类）。
- 选择某标签后，只显示 `item.HasTag(标签)` 为真的道具。
- 显示顺序不重算，仍按背包当前排序序列显示。

## 5. 鼠标悬浮窗显示哪些信息

### 5.1 当前默认悬浮字段（未自定义时）

- `name`
- `description`
- `acquisition_hint`
- `base_rarity`
- `sell_price`
- `tags`

### 5.2 如何为单个物品自定义悬浮信息

在该物品 YAML 中写 `hover_info_fields`，按你写的顺序展示。  
可用字段 ID：

- `item_id`
- `parent_id`
- `category`
- `name`
- `description`
- `detail_description`
- `acquisition_hint`
- `quantity`
- `base_rarity`
- `current_rarity`
- `base_max_stack`
- `buy_price`
- `sell_price`
- `has_durability`
- `max_durability`
- `current_durability`
- `tool_log_yield`
- `tool_chop_speed`
- `tags`
- `is_favorite`
- `is_junk`
- `arrival_order`
- `display_order`

若要附加描述文本，可加 `hover_extra_lines`。

## 6. 详细属性栏接口（已预留）

### 6.1 当前机制

- 每个物品可通过 `detail_info_fields` 指定详细栏显示字段顺序。
- 若未配置，则走系统默认详细字段集合。
- 可通过 `detail_extra_lines` 附加说明性文本。

### 6.2 供后续新 UI 直接调用的接口

- `InventoryPanel.GetItemDetailEntries(string itemId)`  
  返回该道具当前“详细属性页”可见字段列表（已含数量、运行态状态等）。
- `InventoryPanel.ItemDetailFocused` 事件  
  鼠标进入道具格时抛出该道具的详细字段列表，便于你后续把右侧详细栏直接联动上去。

---

## 7. 示例：数值型与描述型分别配置

```yaml
items:
  - id: "tool_fishing_rod"
    ...
    hover_info_fields: ["name", "description", "quantity", "base_rarity", "sell_price", "tags"]
    detail_info_fields: ["item_id", "name", "description", "quantity", "base_rarity", "base_max_stack", "sell_price", "tags"]

  - id: "quest_letter_xxx"
    ...
    hover_info_fields: ["name", "description", "detail_description"]
    hover_extra_lines:
      - "这是一封关键任务道具，请勿出售。"
    detail_info_fields: ["item_id", "name", "description", "detail_description", "acquisition_hint", "tags"]
```
