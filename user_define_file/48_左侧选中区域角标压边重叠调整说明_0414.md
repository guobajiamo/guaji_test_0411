# 48_左侧选中区域角标压边重叠调整说明_0414

## 1. 本次调整目标

用户反馈左侧“当前所在二级区域”的橙色角标位置不对。

之前的角标被放在蓝色选中底块内部，看起来像是缩进去的一层描边，不够自然。

本次调整为：

- 让橙色角标贴住蓝色底块外缘
- 让角标与蓝色背景边缘产生少量重叠
- 保留原有四角小括号式提示，不改成整圈外框

---

## 2. 实际修改内容

涉及文件：

- `Scripts/UI/GamePlayUI.Layout.cs`
- `Scripts/UI/GamePlayUI.Actions.cs`

### 2.1 选中项层级抬高

在 `BuildSecondaryAreaNode(...)` 中，将选中项面板的 `ZIndex` 提高：

- 未选中：`0`
- 选中：`2`

作用：

- 让选中项四角橙色角标在视觉上压住邻近空隙
- 避免角标伸出面板边缘后被相邻节点遮住

### 2.2 角标参数改为“压边式”

原本调用为：

`CreateCornerBracketOverlay(new Color("#f6a53b"), 16, 2, 2)`

现改为：

`CreateCornerBracketOverlay(new Color("#f6a53b"), 18, -2, 3)`

含义：

- `cornerSize = 18`
  角标长度略增，更容易看清
- `inset = -2`
  角标不再缩进，而是向外压出 2 像素
- `thickness = 3`
  角标线条更饱满，不会显得细弱

### 2.3 圆角贴合当前蓝色底块

在 `CreateCornerBracketPiece(...)` 中，不再写死较小圆角，而是改为参考主题统一圆角：

- `int bracketRadius = Math.Max(_theme.CornerRadius, 10);`

作用：

- 让橙色角标弧度更贴近当前左栏选中底块的圆角
- 视觉上更像贴边包住，而不是单独悬浮的小线段

### 2.4 角标覆盖层层级提高

在 `CreateCornerBracketOverlay(...)` 与 `CreateCornerBracketPiece(...)` 中补充较高 `ZIndex`。

作用：

- 保证角标在选中项内部内容之上稳定显示
- 让压边部分更不容易被其它控件干扰

## 3. 如果后续还想继续微调，改哪里

主要入口：

- `Scripts/UI/GamePlayUI.Layout.cs`
  `BuildSecondaryAreaNode(...)`

重点参数：

- `CreateCornerBracketOverlay(color, cornerSize, inset, thickness)`

其中：

- `cornerSize` 控制角标长度
- `inset` 控制角标是缩进还是外压
- `thickness` 控制线条粗细

辅助入口：

- `Scripts/UI/GamePlayUI.Actions.cs`
  `CreateCornerBracketPiece(...)`

重点参数：

- `bracketRadius`
  控制四角括号的弧度大小
