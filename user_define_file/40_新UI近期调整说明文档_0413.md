# 40_新UI近期调整说明文档_0413

## 1. 本轮调整目标

这一次不是重新换整套 UI 架构，而是在当前已经基本满意的深色分区配色基础上，继续把几个具体显示问题修顺：

- 子场景标题条颜色需要回退到更中性的浅灰半透明样式
- 整体区块之间的缝隙略大，希望更紧凑
- 底部 LOG 区需要从“两段状态”改成“三段状态”
- 右侧人物面板里的技能栏需要补充等级与可升级提示

本次仍然遵守原有原则：

- 不改业务层主要结构
- 不改 UI 主体架构
- 只在 UI 展示层和轻量交互层做修正

---

## 2. 子场景标题条调整

调整文件：

- `Scripts/UI/GamePlayUI.Layout.cs`

本次修改：

1. 把子场景标题条的底色改回接近早期版本的样式  
   使用 `30%` 透明度的白色半透明条，而不是现在这版偏彩色的标题条。

2. 把标题条高度压低  
   通过减小标题条上下边距，让它只比文字高度略高，不再显得厚重。

3. 稍微收紧子场景卡片内部标题与正文之间的间距  
   让整个子场景卡片更利落。

这对应你提到的“琉璃点心屋标题条太高、颜色不合适”的问题。

---

## 3. 整体边框与缝隙调整

调整文件：

- `Scripts/UI/GamePlayUI.Actions.cs`
- `Configs/UI/gameplay_ui_theme.yaml`

本次修改：

1. 主面板统一边框从之前的较细宽度加粗  
   让边框更有存在感，也更能吃掉不同框体之间的缝隙感。

2. 标签按钮边框也同步略加粗  
   避免中间标签页显得过薄。

3. 全局 `section_gap` 与 `inner_gap` 略微收紧  
   让左中右区域、日志区、信息区之间整体更紧凑。

目的：

- 不通过砍掉结构来变紧，而是通过“边框更实 + 间隙略缩”来让界面更完整

## 4. 底部 LOG 区三段式状态

调整文件：

- `Scripts/UI/GamePlayUI.cs`
- `Scripts/UI/LogPanel.cs`

### 4.1 修改前

原先底部 LOG 区只有两种状态：

1. 默认 5 行高
2. 点击“展开”后，弹出一个更大的悬浮窗显示完整日志

### 4.2 修改后

现在改成三段式状态：

1. 默认状态  
   显示约 5 行日志高度

2. 最小化状态  
   点击默认状态下新增的“最小化”按钮后，LOG 缩到单行高度

3. 最大化状态  
   点击“最大化”按钮后，弹出大悬浮窗显示更多日志

### 4.3 新按钮逻辑

默认 5 行高状态下：

- 原“展开”按钮重命名为“最大化”
- 在它左边新增一个“最小化”按钮

处于最小化状态时：

- 原“最小化”按钮文字变为“展开”
- 点击这个“展开”按钮后，恢复回默认 5 行高

处于最大化浮窗状态时：

- 点击“收起”按钮后
- 关闭浮窗
- 并且底部 LOG 回到默认 5 行高状态，而不是停留在最小化状态

### 4.4 具体实现方式

`LogPanel` 现在支持：

- 展开模式：显示全部日志
- 折叠模式：按指定行数截取最近日志

因此当前底部 LOG 区已经支持：

- 1 行
- 5 行
- 全量展开

三种显示层级。

## 5. 右侧技能栏显示增强

调整文件：

- `Scripts/UI/CharacterStatusPanel.cs`

### 5.1 修改前

技能栏里每个技能基本只显示技能名称。

### 5.2 修改后

现在每个技能行会显示：

- 左侧：技能名称
- 右侧：当前技能等级，例如 `Lv.3`
- 如果当前经验值已经足够手动升级，则在等级右边额外显示红色小字：
  `(可升级)`

### 5.3 说明

为了保留当前结构而不重做整个人物面板，本次没有把技能栏改成复杂卡片，而是在现有折叠分组内，把每一行技能条目扩展成：

- 左侧主信息
- 右侧等级信息
- 右侧小号升级提示

同时原本的 tooltip 详细说明仍然保留，包括：

- 技能描述
- 当前等级
- 当前经验
- 总经验
- 适用工具

## 8. 这份文档之后有什么用

如果后面继续调 UI，建议优先回看这份 `40` 号文档，因为它记录了当前几处最容易被后续改动打乱的细节：

1. 子场景标题条必须保持“浅灰半透明、低高度”的风格
2. LOG 区已经不是两段式，而是“最小化 / 默认 / 最大化”三段式
3. 右侧技能栏已经带有等级和可升级提示，不要后续误改回只显示技能名
4. 边框加粗和整体收紧已经是当前 UI 紧凑感的一部分

也就是说，后续如果再继续调色或改排版，这几个点最好保留，不然很容易把这轮修顺的观感又改乱。
