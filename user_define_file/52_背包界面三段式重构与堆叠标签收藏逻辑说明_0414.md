## 3. 收藏功能与重排规则

已按需求落地：

1. 收藏某物品：  
该物品立即移动到背包最前（第一排第一位），并刷新一次。  

2. 多收藏顺序：  
最后一次收藏的物品排在最前，之前收藏物品依次后移。  

3. 取消收藏：  
该物品从收藏区移出，并插入到“非收藏物品区”的第一位。  

4. 约束：  
任何情况下都保证“收藏物品整体在前，非收藏物品在后”。

## 4. 物品“是否可堆叠”能力

### 4.1 新增字段

- `ItemDefinition.IsStackable`（代码字段）
- YAML 字段：`is_stackable`

### 4.2 展示行为

- `is_stackable: true`：同物品按数量合并显示（`名称(总数)`）。
- `is_stackable: false`：同物品按单格占位显示（一个数量占一格）。

> 当前运行态库存底层仍是按物品 ID 聚合计数，但背包展示层已按“不可堆叠单格占位”规则展开显示。

### 4.3 已回填现有 YAML

已对当前场景物品定义回填 `is_stackable`：

- 货币/素材/消耗品：设为 `true`
- 工具/战斗相关：设为 `false`
