# 22_新UI的图片配置说明文件

## 1. 当前 UI 方案的核心结论

本轮已经把游戏 UI 的设计基准固定为 `1920x1080`。

当前实际生效的关键位置如下：

- 分辨率基准：`project.godot`
- 布局参数：`Resources/UI/main_ui_layout.tres`
- 窗口缩放逻辑：`Scripts/UI/AppRoot.cs`
- 图片主题配置：`Configs/UI/ui_image_theme.yaml`
- PNG 素材目录：`Assets/UI/Images/`

当前缩放策略不是“无限放大铺满”，而是：

- 以 `1920x1080` 作为设计稿基准
- 窗口缩小时，整套 UI 按比例压缩
- 窗口放大时，不主动放大到超过 1 倍

这样做的目的，是优先避免你后续提供的 PNG UI 被放大后发糊。

## 2. 现在可以通过 YAML 配置哪些图片

配置文件路径：`Configs/UI/ui_image_theme.yaml`

目前已经接入了这些图片入口：

### 2.1 主菜单背景

```yaml
menu_background:
  fallback_color: "#11161d"
  image_path: "res://Assets/UI/Images/MainMenu/main_menu_background.png"
```

说明：

- `fallback_color` 是没有图片时的默认暗色背景
- `image_path` 指向主菜单背景 PNG
- 如果图片不存在，会自动退回到暗色背景

### 2.2 主菜单按钮

当前已经支持：

- `menu_primary`
- `menu_test_entry`

对应的是：

- 主菜单里的“新的游戏 / 读取存档 / 游戏设置 / 退出游戏”
- 右上角绿色的“进入游戏测试”按钮

### 2.3 事件按钮

当前已经支持：

- `event_oneshot`
- `event_click`
- `event_idle`

它们分别对应：

- 一次性事件按钮
- 单次点击事件按钮
- 挂机事件按钮

### 2.4 其他按钮

当前还顺手接入了：

- `system_action`
- `system_return_menu`
- `special_action`

它们分别对应：

- 正式剧本“系统”标签里的保存/读取按钮
- “返回主菜单”按钮
- 测试剧本左下角的快捷保存/读取按钮

## 3. 每种按钮图片怎么写

每种按钮样式都支持 4 张状态图：

- `normal_image`
- `hover_image`
- `pressed_image`
- `disabled_image`

同时也支持纯颜色回退：

- `fallback_normal_color`
- `fallback_hover_color`
- `fallback_pressed_color`
- `fallback_disabled_color`
- `text_color`
- `disabled_text_color`
- `border_color`

示例：

```yaml
buttons:
  event_click:
    normal_image: "res://Assets/UI/Images/Buttons/event_click_normal.png"
    hover_image: "res://Assets/UI/Images/Buttons/event_click_hover.png"
    pressed_image: "res://Assets/UI/Images/Buttons/event_click_pressed.png"
    disabled_image: "res://Assets/UI/Images/Buttons/event_click_disabled.png"
    fallback_normal_color: "#1d3557"
    fallback_hover_color: "#24466f"
    fallback_pressed_color: "#162840"
    fallback_disabled_color: "#596779"
    text_color: "#f1f5f9"
    disabled_text_color: "#d7dee7"
    border_color: "#0d1a2a"
```

如果你暂时还没画完图片，可以只改颜色，等 PNG 做好后再把路径填进去。

## 4. PNG 素材应该放到哪里

当前已经建立好的目录：

- `Assets/UI/Images/MainMenu/`
- `Assets/UI/Images/Buttons/`

建议你后续继续按这个习惯放：

- 主菜单背景图放 `Assets/UI/Images/MainMenu/`
- 各类按钮图放 `Assets/UI/Images/Buttons/`

如果以后你还想扩展别的 UI 图片，也可以在 `Assets/UI/Images/` 下继续细分子目录。

## 5. 推荐的命名方式

建议保持和 YAML key 对应，这样最好查：

- `main_menu_background.png`
- `menu_primary_normal.png`
- `menu_primary_hover.png`
- `menu_primary_pressed.png`
- `menu_primary_disabled.png`
- `event_oneshot_normal.png`
- `event_click_normal.png`
- `event_idle_normal.png`

这样你一看文件名，就知道它是给哪一类 UI 用的。

## 6. 图片拉伸与适配规则

当前主菜单背景图片会按“尽量铺满画面”的方式显示。

事件按钮和菜单按钮则不会做复杂九宫格切片，而是采用较直接的按钮贴图方式：

- 有图片时，优先使用图片
- 没图片时，退回颜色样式

这意味着你现在最适合先准备：

- 尺寸比较接近目标按钮大小的 PNG
- 或者先只用颜色把流程跑通

如果你后面希望按钮图片支持更精细的九宫格拉伸，我可以下一轮再帮你把 `StyleBoxTexture` 改成带边距切片的版本。

## 7. 你现在改图的最短路径

如果你现在就要测试新图，建议按这个顺序来：

1. 把主菜单背景图放进 `Assets/UI/Images/MainMenu/main_menu_background.png`
2. 把主菜单按钮图放进 `Assets/UI/Images/Buttons/`
3. 打开 `Configs/UI/ui_image_theme.yaml`
4. 检查路径是不是 `res://` 开头
5. 重新运行 Godot

只要路径正确，程序会自动尝试加载这些 PNG。

## 8. 当前需要你特别注意的一点

现在真正被程序读取的 UI 图片配置文件只有这一个：

- `Configs/UI/ui_image_theme.yaml`

也就是说，你后面如果想替换图片，不需要去改 C# 逻辑，优先改这个 YAML 就行。
