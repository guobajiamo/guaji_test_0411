《UI 调整规则说明》

这份说明书的目标是：
尽量让你以后调这款文字游戏的 UI 时，优先改“参数”和“资源”，而不是一上来就去改大段业务代码。


================================
一、先记住当前 UI 的实现方式
================================

当前主界面不是在 `MainUI.tscn` 里手工摆满所有控件，
而是采用了下面这种方式：

`Scenes/UI/MainUI.tscn`
只保留一个轻量根节点
-> 由 `Scripts/UI/MainUI.cs` 在运行时动态创建整套界面
-> 布局参数来自 `Resources/UI/main_ui_layout.tres`

这意味着：

1. 你想调“边距、列宽、字体、按钮高度、刷新间隔”
   优先改 `Resources/UI/main_ui_layout.tres`

2. 你想调“左中右三列里放什么面板、有哪些标签页”
   需要改 `Scripts/UI/MainUI.cs`

3. 你想调“某个面板里面显示什么内容”
   去改对应面板脚本，例如：
   - `Scripts/UI/InventoryPanel.cs`
   - `Scripts/UI/SkillPanel.cs`
   - `Scripts/UI/CharacterStatusPanel.cs`
   - `Scripts/UI/FactionPanel.cs`


================================
二、最推荐的调整入口
================================

最推荐你先改这个文件：

`Resources/UI/main_ui_layout.tres`

这个资源文件对应的脚本类是：

`Scripts/UI/MainUiLayoutSettings.cs`

也就是说：
你在 Godot 编辑器里点开 `main_ui_layout.tres`，
Inspector 面板里看到的那些字段，
就是 `MainUiLayoutSettings.cs` 里 `[Export]` 出来的参数。


================================
三、每个可调参数是什么意思
================================

1. `OuterMargin`
作用：
- 控制整个主界面距离窗口边缘的外边距

你可以这样理解：
- 数值越大，整体 UI 越往中间缩
- 数值越小，整体 UI 越贴近屏幕边缘

适合什么时候调：
- 觉得画面太挤
- 想让边缘留白更多


2. `PanelSpacing`
作用：
- 控制各个大面板之间的间距

你可以这样理解：
- 数值越大，左中右列、上下区块之间越松
- 数值越小，内容会更紧凑


3. `LeftColumnMinWidth`
作用：
- 控制左侧按钮列的最小宽度

当前左侧放的是：
- 一次性事件
- 点击事件
- 挂机事件

如果你觉得按钮文字容易换行，或者想让左侧更宽，就调大它。


4. `RightColumnMinWidth`
作用：
- 控制右侧状态栏列的最小宽度

当前右侧放的是：
- 上手提示
- 角色状态

如果你觉得状态文字太挤，就调大它。


5. `HeaderFontSize`
作用：
- 控制页面最顶部大标题字号


6. `SectionHeaderFontSize`
作用：
- 控制左侧各个按钮区标题字号
- 例如“一次性事件”“点击事件”“挂机事件”


7. `BodyFontSize`
作用：
- 控制大部分正文文字字号
- 包括提示文字、按钮区空状态文字等


8. `EventButtonMinHeight`
作用：
- 控制左侧事件按钮的最小高度

如果你想让按钮更厚、更容易点，就调大它。


9. `RefreshIntervalSeconds`
作用：
- 控制 UI 刷新频率

当前意义：
- 数值越小，界面刷新越勤，进度显示越及时
- 数值越大，刷新越省性能，但进度更新看起来会更“跳”

建议：
- 新手先保持默认 `0.2`
- 如果后续界面元素越来越多，可以尝试改到 `0.3 ~ 0.5`


================================
四、最安全的 UI 调整方式
================================

如果你只是想改“排版”和“大小”，建议按这个顺序来：

第 1 步：
打开 `Resources/UI/main_ui_layout.tres`

第 2 步：
先只调一个参数

第 3 步：
回到 Godot 运行场景查看效果

第 4 步：
确认没问题后，再继续调下一个参数

为什么建议一次只改一个？
因为你是代码初学者时，这样最容易知道“到底是哪一个参数造成了画面变化”。


================================
五、如果你想换一套布局资源
================================

`Scripts/UI/MainUI.cs` 里有一个可导出的路径字段：

`LayoutSettingsPath`

默认值是：

`res://Resources/UI/main_ui_layout.tres`

这表示：
主界面启动时，会从这条路径读取布局资源。

所以你以后完全可以：

1. 复制一份新的 `.tres`
   例如：
   `Resources/UI/main_ui_layout_mobile.tres`

2. 在 Godot 里把 `MainUI` 节点的 `LayoutSettingsPath` 改成新资源路径

这样你就可以做：
- PC 版布局
- 窄屏布局
- 更大的字体版布局

而不需要去改 `MainUI.cs` 的业务逻辑。


================================
六、哪些调整必须改代码
================================

以下情况，改 `.tres` 不够，需要改脚本：

1. 想新增一个新的左侧按钮分组
要改：
- `Scripts/Core/Enums/ButtonListGroup.cs`
- `Configs/Events/*.yaml`
- `Scripts/UI/MainUI.cs`

2. 想新增一个新的标签页
要改：
- `Scripts/UI/MainUI.cs`
- 新标签页对应的面板脚本

3. 想改变某个面板里的显示内容
要改对应面板脚本，例如：
- 背包：`Scripts/UI/InventoryPanel.cs`
- 技能：`Scripts/UI/SkillPanel.cs`
- 日志：`Scripts/UI/LogPanel.cs`
- 状态：`Scripts/UI/CharacterStatusPanel.cs`

4. 想给界面加图片、图标、特殊颜色主题
优先会涉及：
- `Resources/Themes/default_theme.tres`
- 对应 UI 脚本里的控件创建逻辑


================================
七、一个很重要的注意事项
================================

因为当前主界面的大部分控件是运行时动态创建的，
所以你如果直接去 `Scenes/UI/MainUI.tscn` 里手工加很多子节点，
它们不一定会自动参与当前这套逻辑。

最稳妥的原则是：

1. 改排版参数：优先改 `main_ui_layout.tres`
2. 改布局结构：去改 `MainUI.cs`
3. 改面板内容：去改对应面板脚本


================================
八、给你的一套简单上手建议
================================

如果你是第一次自己调这套 UI，我建议先这样试：

1. 把 `LeftColumnMinWidth` 从 `320` 改成 `360`
2. 把 `RightColumnMinWidth` 从 `280` 改成 `320`
3. 把 `BodyFontSize` 从 `16` 改成 `18`
4. 把 `EventButtonMinHeight` 从 `54` 改成 `64`

这样你会很直观地看到：
- 左右列变宽
- 字体变大
- 按钮更厚

这会帮助你快速理解“这些参数分别管什么”。


================================
九、最后一句话总结
================================

当前这版 UI 的设计思路是：

“布局优先参数化，逻辑优先代码化。”

也就是说：
常见的 UI 微调，尽量通过 `Resources/UI/main_ui_layout.tres` 解决；
只有当你想改变界面结构和功能时，才去改 `MainUI.cs` 和各个面板脚本。
