# 28_游戏内容界面新UI重构说明文档_0413

## 1. 本次改动目标

这次重构的范围只针对游戏内容界面，主菜单界面保持不变。

本次完成的重点有：

- 将原本左侧三列事件按钮布局，重构为“左侧区域栏 + 中央当前区域/标签页 + 底部日志 + 右侧信息/状态”的新框架
- 将原本居中的“区域”标签页独立到左侧竖栏
- 删除中部“势力”标签页
- 新增中部第一个固定标签页“当前区域”
- 将测试剧本也统一接入“系统”标签页
- 移除测试剧本底部单独的一键存档/读档按钮
- 右侧下半部人物状态栏改为“目标 / 技能 / 状态 / 货币 / 声望”
- 新增可配置的 UI 主题 YAML 与剧本区域布局 YAML
- 新增场景收藏、当前区域记忆、场景收藏顺序存档

---

## 2. 当前界面结构

现在游戏内容界面的结构是：

1. 顶部菜单栏  
   显示当前剧本与当前区域

2. 左侧区域栏  
   显示一级地区、二级区域、展开折叠、可互动数量、挂机区域红色星标

3. 中央视图区域  
   顶部是自定义标签栏  
   下方是标签内容页  
   第一个页签固定是“当前区域”

4. 底部日志栏  
   只占中部宽度，不再横跨左右栏

5. 右侧上半部信息面板  
   继续复用之前已经做好的 `info_panel.yaml` 配置方案

6. 右侧下半部人物状态栏  
   改为“目标 / 技能 / 状态 / 货币 / 声望”五个可折叠大类

## 5. 当前运行规则

### 5.1 左侧区域栏

- 一级地区支持展开与折叠
- 二级区域点击后，会立刻切换中部到“当前区域”页
- 二级区域右侧绿色小字显示“当前可互动按钮数”
- 如果当前挂机事件属于该区域，则左侧显示红色 `★`

### 5.2 当前区域页

- 每个场景独立成卡片
- 场景支持右键收藏
- 收藏场景会置顶，且最新收藏的排在更上方
- 场景内部按钮按：
  - 一次性事件
  - 单次点击事件
  - 挂机事件
  的顺序排序
- 每行最多两列按钮
- 挂机按钮上方显示进度条
- 正在执行中的挂机按钮前会显示红色 `★`

### 5.3 标签页

当前保留的中部标签页是：

- 当前区域
- 背包
- 技能
- 图鉴
- 系统

当前已删除的旧标签页：

- 势力
- 区域

### 5.4 系统页

- 正式剧本：系统页显示槽位保存 / 槽位读取 / 返回主菜单
- 测试剧本：系统页显示一键存档 / 一键读档 / 返回主菜单

也就是说，现在测试剧本和正式剧本终于使用同一个系统页入口，只是按钮行为不同。

### 5.5 右侧状态栏

现在右侧状态栏分为：

- 目标
- 技能
- 状态
- 货币
- 声望

其中：

- “目标”栏会显示当前是否正在挂机
- 如果正在挂机，会显示一个小进度条
- 技能详细信息、等级、经验、适用工具，都收进对应技能项的 Tooltip 里
- 旧的“工具”和“当前挂机项目”独立大类已经移除

## 8. 你下次如果继续改这套 UI，建议先看哪里

推荐先按这个顺序看：

1. `Configs/UI/gameplay_ui_theme.yaml`
2. `Configs/Scenarios/对应剧本/UI/gameplay_layout.yaml`
3. `Scripts/UI/GamePlayUI.cs`
4. `Scripts/UI/GamePlayUI.Layout.cs`
5. `Scripts/UI/GamePlayUI.Actions.cs`
6. `Scripts/UI/CharacterStatusPanel.cs`

如果只是调颜色、字号、边距、透明度，优先改 YAML。

如果只是加一个新区域、新场景、新按钮归属，优先改 `gameplay_layout.yaml`。

如果要改交互规则、布局结构、收藏逻辑、区域切换逻辑，再改 C#。
