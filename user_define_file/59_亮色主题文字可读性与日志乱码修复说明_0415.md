# 59_亮色主题文字可读性与日志乱码修复说明_0415

## 1. 本轮问题
- 亮色新UI（stitch）下，部分文字仍沿用深色旧UI的浅色字体，导致“白字叠浅底”辨识度低。
- 底部日志栏标题与按钮文案出现乱码。

## 2. 已完成修复

### 2.1 当前场景 > 子场景标题（按主题切换文字色）
- 文件：`Scripts/UI/GamePlayUI.Layout.cs`
- 调整点：`BuildSceneCard(...)`
  - 子场景标题顶部高光线、底部阴影线、三层标题字色全部改为 `IsUsingStitchUiTheme()` 分支。
  - stitch 下改为深色可读方案，legacy 保持原先深色UI方案。

### 2.2 背包 > 分类标签默认/选中颜色（按主题切换）
- 文件：`Scripts/UI/InventoryPanel.cs`
- 调整点：`RefreshFilterButtons(...)`
  - 分类按钮颜色改为按状态解析后统一覆盖 `font_color / font_hover_color / font_pressed_color`。
  - stitch 下强化默认与选中对比：
    - active: `#0f4f4c`
    - inactive: `#7b8a84`

### 2.3 背包右侧物品详情文字（按主题切换）
- 文件：`Scripts/UI/InventoryPanel.cs`
- 调整点：`BuildMainArea()`
  - 为右侧 `RichTextLabel` 明确设置 `default_color`，避免亮色主题下继承白字。
  - stitch：
    - 标题 `#2f3b39`
    - 正文 `#4f5f5b`
  - legacy 保持偏亮色文本方案。

### 2.4 底部日志栏乱码修复
- 文件：`Scripts/UI/GamePlayUI.cs`
- 修复文案：
  - `运行日志`
  - `最小化`
  - `最大化`
  - `收起`
  - `展开`
- 同步修复了折叠状态切换时按钮文本更新逻辑中的乱码字串。

## 3. 验证结果
- `dotnet build .\\test_00_0410.csproj`：通过（0 error / 0 warning）。
- `UiFeatureSmokeTest.tscn` headless 冒烟：通过，日志未见脚本报错。

## 4. 同类问题排查结论
- 已确认：代码中仍存在“疑似同类乱码字面量”的历史残留（主要分布在 `GamePlayUI.Layout.cs`、`GamePlayUI.Actions.cs`、`InventoryPanel.cs` 的部分提示与日志文案路径）。
- 这些残留不一定全部在当前主流程立即可见，但存在后续触发风险；建议下一轮做一次“全UI字面量统一清洗”。
