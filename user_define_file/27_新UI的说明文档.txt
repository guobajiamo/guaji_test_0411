总体布局配色说明：
游戏整体背景为纯黑底色。左侧、中央区域、底部区域、右侧区域的这几个不同区域中，需要使用25%透明度的不同底色对这些区块做出分割，方便辨识。与此同时配色需要好看，且按钮上的文字或直接呈现在区块背景上的文字均需考虑对比度方便辨识。
在最后输出的说明文档中，给我提供一个ymal脚本，供我修改每个区域的文字、按钮、区域色块颜色、区域色块透明度等等所有UI数值化相关的参数供我调节。
下面进行每个版块的设计说明：
这套UI中，总共分为‘顶部菜单栏’，‘左侧区域栏’，‘中央标签栏’，‘底部日志栏’，‘右侧上半部信息面板’，‘右侧下半部人物状态栏’这些不同区域，有部分不仅改动了UI显示，同时还带出了新的功能需求，下面我逐一讲解说明。

1. 左侧竖列‘区域’栏：
这套新UI中，将原本放在‘标签’这一栏中的‘区域’独立出来，放到了屏幕的左侧区域做成了单独一列，而这一类区域中也分为不同等级和类型的区域。
一级地区标签：如我的图示中所示的几个大类区域，‘人间之里、魔法之森、博丽神社、妖怪之山’显示为大号绿色，而特殊的可以获取阵营声望的势力如‘红魔馆’则显示为大号红色。一级地区标签需提供展开与折叠功能（既小号字体的[-]和[+]）。
二级地区标签：如人间之里下的商店街、自警队办事处、稗田邸这类无战斗副本的区域均显示为深黄（橙）色，在这里通常会有各种事件。而如‘魔法之森’下的‘魔法之森外围’和‘魔法之森深处’则显示为中号深红色字，标致这场景下有战斗副本。
势力阵营所属的二级标签是特殊的，因为一个势力中可能存在多个可访问区域，因此我考虑将这些二级区域以‘一行显示两个区域’的方式排布以减少下拉菜单条的长度。但这只是个设想，本次可以为这种设计留出接口但无需实现（本次UI重置左侧默认一行只显示1个区域）。
特别说明，在每个二级区域中，如果该区域内部的‘可互动按钮’大于0，那么需要在区域右侧以小号绿色文字进行标注。什么是‘可互动按钮’？既当下玩家已经满足点击条件（例如满足提交物资的库存条件，因此点亮了‘一次性提交事件’）的按钮的统称。除了满足条件的一次性事件外，满足条件的（例如消费金额达标）的单次点击按钮，满足技能条件（例如至少获得一把石斧）的挂机按钮（砍树）都算可互动按钮。
以中央区域展示的‘商店街’为例，琉璃点心屋场景下有4个可互动按钮，甘味茶屋有1个可互动按钮，孫惣刃物锻冶有3个可互动按钮，因此左侧二级区域标签‘商店街’的右侧需要有一个<size=50%>（8）</size>的小号绿色文字来提示玩家‘这个场景有按钮可以点’。

2. 中央标签栏
原本中央的标签栏中的‘区域’现在被独立出来放到了左侧做成了竖边栏，现在需要删除‘势力’这个标签，并在背包的左侧，也就是标签的最左侧新增一个‘当前区域栏’，任何情况下都会存在一个当前区域，在进入新剧本时也会直接进入左侧区域栏中的第一个区域。如果玩家点击‘新的游戏’第一次进入剧本，也会默认激活左侧第一个二级区域。
每个二级区域中，存在多个场景，每个场景中有不同类型的事件组合。以示例图中的‘商店街’二级区域为例，该区域中存在着三种不同场景：‘● 琉璃点心屋’场景被置顶显示，意味着该场景已经被玩家右键场景标签进行了‘收藏’，被收藏的场景会从默认排序位置提升至所有场景的最顶部位置。如果玩家收藏了多个场景，那最新收藏的场景位于最顶上，依次排序。
每个场景具备一个独立的自适应的框体区域，区域内部需要正好包含当前可互动的所有按钮。一个场景下可以包含任意多个按钮，但每一列只能放两个按钮，不同类型的按钮按优先级排序：‘一次性事件按钮（排序优先级高）’、‘单次点击事件按钮（排序优先级中）’、‘挂机事件按钮（排序优先级低）’，需要按次序进行排序。如图例中的‘琉璃点心屋’场景下，‘委托·提交食材（1/5）’和‘打听消息’这两个一次性事件排在第一行，‘购买三色团子（1个）’排在第二行的前面，而挂机事件‘打工’则排在四个事件的最后（第二行后面）。同时，需要对这三类不同事件的按钮做出区分：一次性事件为红色按钮，单次点击事件为绿色按钮，挂机事件为深黄色（橙色）按钮。
特殊说明，当玩家点击选择了可互动的‘挂机事件’后，需要在左侧区域栏对应的二级区域标签的左侧显示出一个红色五角星★符号，来告诉玩家角色当前在该区域执行挂机活动。而在对应的挂机按钮上也要出现该红色五角星符号。与此同时，我需要在紧贴着挂机按钮的顶部实例化一个进度条来显示挂机进度（替代早期版本的文字百分比显示，让画面更直观）。
最后，关于其他标签项：将当前正在激活的标签标志放大并高亮标签颜色（如PNG示例图中当前的标签页正在第一个‘当前区域’标签页下，因此该标签页被放大并且高亮），如果切换到其他标签，也要做类似的处理（先前放大的标签在它不再被激活时缩放至与其他标签相同大小）。
额外注意：当玩家选择从左侧竖边栏点击选择了一个具体的二级地区时，中央的标签也需要立刻切换到第一个‘当前区域’标签，并将区域内的内容刷新为对应二级区域所属的场景内容。

‘右侧上半部信息面板’与先前预留的可以通过ymal接口配置的版本保持不变。
‘右侧下半部人物状态栏’的UI出现了新的变化，将先前的‘工具’和‘当前挂机项目’这两个大类去掉了。现在的可折叠大类从上到下排序依次是‘目标’、‘技能’、‘状态’、‘货币’、‘声望’。关于当前所持石斧、以及技能等级这些信息，全部汇总显示在对应技能的‘鼠标指针焦点悬浮窗’中，不再额外单独列出。
值得一提的是，在‘目标’栏，如果玩家当前没有任何挂机项目，那么这里会显示‘正在休息’。而如果如图中所示，正在‘商店街’的‘琉璃点心屋’执行‘打工’这个挂机事件来赚钱，那么这里就会显示‘正在琉璃点心屋打工’，与此同时要在顶部显示一个小号的进度条（如果玩家切换到背包或者战斗界面，他可以从右侧的目标栏确认自己是否正在正常挂机）。
