## 4. 你接下来需要做的配置步骤

下面建议你按顺序处理。

### 4.1 安装 Android Studio

最推荐的方式是直接安装 Android Studio。

原因：

- 它自带 SDK Manager
- 安装 Android SDK 最省事
- 后续维护平台版本也方便

如果你已经安装过 Android Studio，但路径改过或删过，也建议重新确认其 SDK 设置。

### 4.2 使用 SDK Manager 安装 Android SDK 关键组件

至少要安装下面这些组件：

- `Android SDK Platform-Tools`
- `Android SDK Build-Tools`
- 至少一个 `Android SDK Platform`
- `Android SDK Command-line Tools`

建议先装一个较新的稳定 Android 平台版本。

### 4.3 确认 Android SDK 根目录

建议统一使用：

```text
C:\Users\AA\AppData\Local\Android\Sdk
```

如果你想用其他目录也可以，但之后：

- Godot 设置
- 环境变量

都要和这个目录保持一致。

### 4.4 配置环境变量

建议至少配置：

```text
JAVA_HOME=C:\Program Files\Microsoft\jdk-21.0.1.12-hotspot
ANDROID_SDK_ROOT=你的 Android SDK 根目录
```

并把下面这个目录加入 `Path`：

```text
你的 Android SDK 根目录\platform-tools
```

配置完成后，重新打开终端，检查：

```powershell
java -version
adb version
```

如果这两个都正常，说明系统层环境基本到位。

### 4.5 在 Godot 中填写 Android 导出设置

打开 Godot 4.6.2 Mono，进入：

```text
Editor Settings -> Export -> Android
```

把下面两个路径填好：

- `Java SDK Path`
- `Android SDK Path`

建议填写为：

- `Java SDK Path = C:\Program Files\Microsoft\jdk-21.0.1.12-hotspot`
- `Android SDK Path = 你的 Android SDK 根目录`

### 4.6 生成 debug keystore

如果只是做手机测试，先用 debug keystore 就够了。

你后续可以：

- 让 Godot 自动生成
- 或者再让我帮你确认/生成

但总之最后需要让这个文件真实存在。

### 4.7 打开手机开发者模式和 USB 调试

如果你要把 APK 安装到手机上进行真机测试，需要：

- 打开开发者模式
- 打开 USB 调试

如果后续准备用 `adb install` 安装，这一步是必须的。

### 4.8 给项目新增 Android 导出预设

当前项目里还没有 Android 导出预设，所以你后续还需要在 Godot 里新增：

```text
Project -> Export -> Add -> Android
```

然后配置：

- 包名
- 应用名
- 图标
- 签名
- 架构
- 权限

## 5. 目前最关键的 4 个阻塞点

你现在最优先要解决的是这 4 项：

1. Android SDK 目录要真实存在
2. `adb version` 要能正常执行
3. Godot 中的 Android SDK / JDK 路径要配置正确
4. debug keystore 要真实存在

只要这 4 项没就绪，就不建议直接开始打包 APK。
