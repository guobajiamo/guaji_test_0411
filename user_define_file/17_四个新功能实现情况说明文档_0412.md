- 子项使用稍小字号和蓝色文字。

- 展开后日志会覆盖整个屏幕区域显示。

- 支持单按钮确认型弹窗。
- 支持双选项互斥型弹窗。
- 双选项可以连接到两个不同的一次性事件。
- 当前已在 `oneshot` 事件配置中加入两个演示样例。

## 三、四个功能分别是如何实现的

### 1. 悬浮说明

- 事件按钮现在优先读取 event 的 `hover_info_key`。
- 如果 `hover_info_key` 没填，就回退到 `description_key`。
- 背包物品按钮在 `TooltipText` 中显示：
  `详细描述 + 获取提示 + 稀有度 + 售价`
- 右侧状态栏不再是一个总 `Label`，而是拆成多个 `Control`，所以每一项都可以单独有 `TooltipText`。

### 2. 右侧状态栏折叠分类

- 状态栏内部改成 `ScrollContainer + 多个 PanelContainer`。
- 每个大类标题是一个按钮，点击后会切换展开/折叠。
- 当前写死的四个大类为：`经济`、`工具`、`技能`、`当前挂机项目`。
- “工具”大类会列出玩家当前持有的工具。
- “技能”大类会列出当前技能等级与经验。
- “当前挂机项目”大类会列出当前是否挂机与读条进度。

### 3. 底部日志区

- 主界面不再把“日志”加入中间 `TabContainer`。
- 改为在根布局最底部新增一个日志停靠区。
- 折叠时只取最近 5 行日志显示。
- 同时额外创建一个全屏日志覆盖层。
- 点击“展开”时切到覆盖层，点击“折叠”时返回底部模式。

### 4. 一次性事件弹窗

- `EventDefinition` 新增 `Dialog` 配置。
- `Dialog` 支持两种模式：
  `A. confirm_button_text`：单按钮说明弹窗  
  `B. choices`：双选项分支弹窗
- 主界面点击一次性事件时，会先检查这个事件是否带 `Dialog`。
- 如果有 `Dialog`，就先弹出中央说明框。
- 单按钮模式：点击按钮后真正执行当前事件。
- 双选项模式：点击按钮后执行 `choice.target_event_id` 对应的分支事件。
