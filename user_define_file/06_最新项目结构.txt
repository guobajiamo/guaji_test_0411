《当前项目结构总览（带作用说明）》

阅读建议：
1. 先看“第一部分：目录分工”，快速建立整体印象
2. 再看“第二部分：核心项目树”，理解每个 yaml / tscn / cs 在做什么
3. 最后记住一句话：
   - `Configs/` 放静态配置
   - `Scripts/Core/Definitions` 放静态数据类
   - `Scripts/Core/Runtime` 放玩家运行态
   - `Scripts/Core/Registry` 放注册表
   - `Scripts/Systems` 放业务系统
   - `Scripts/UI` + `Scenes/` 放界面层


================================
一、目录分工（先建立大局观）
================================

根目录：
- `project.godot`：Godot 项目总入口，定义主场景、自动加载脚本等
- `Configs/`：所有 YAML 配置
- `Scripts/`：所有 C# 脚本
- `Scenes/`：所有场景资源（`.tscn`）
- `Resources/`：主题等资源文件
- `Logs/`：当前仓库里保留的开发期日志文件
- `user_define_file/`：需求、架构说明、review 文档和本次输出文档
- `.godot/`：Godot 自动生成的编辑器缓存目录，通常不手改


================================
二、核心项目树（按目录解释）
================================

L:.

├─ `.gitattributes`
  作用：Git 的文本属性设置文件，控制换行符等规则。

├─ `.gitignore`
  作用：Git 忽略规则，通常用来忽略缓存、临时文件、构建输出。

├─ `project.godot`
  作用：Godot 项目配置入口。
  关键内容：
  - 定义项目名
  - 指定主场景 `Scenes/Main.tscn`
  - 指定自动加载脚本 `GameManager`、`SignalBus`

├─ `icon.svg`
  作用：项目图标源文件。

├─ `icon.svg.import`
  作用：Godot 对 `icon.svg` 的导入配置记录。

├─ `.godot/`
  作用：Godot 自动生成的缓存目录。
  说明：这里面的文件大多是编辑器缓存、导入缓存、shader 缓存，不建议手工编辑。

│  ├─ `.gdignore`
  作用：告诉 Godot 某些目录处理方式的辅助文件。

│  ├─ `global_script_class_cache.cfg`
  作用：Godot 记录脚本类信息的缓存。

  作用：Godot 资源 UID 缓存。

  作用：编辑器文件系统缓存。
│  ├─ `script_editor_cache.cfg`
  作用：脚本编辑器缓存。
│  ├─ `editor_layout.cfg`
  作用：编辑器布局缓存。
│  ├─ `editor_script_doc_cache.res`
  作用：脚本文档缓存。
│  ├─ `editor_script_doc_cache.res-folding-...cfg`
  作用：文档折叠状态缓存。
│  └─ `project_metadata.cfg`
  作用：项目编辑器元数据缓存。

│  ├─ `imported/`
  作用：导入资源缓存。
│  ├─ `icon.svg-...ctex`
  作用：图标资源的导入后缓存。
│  └─ `icon.svg-...md5`
  作用：导入资源校验缓存。

│  ├─ `03_v0.4版本claude输出架构（仅声明与备注）.txt`

================================
三、你以后找文件时可以怎么想
================================

如果你以后想：

1. 新增一个物品
- 去 `Configs/Items/items.yaml`
- 若还要补字段结构，再看 `Scripts/Core/Definitions/ItemDefinition.cs`
- 若要查如何进入系统，再看 `ItemRegistry.cs`

2. 新增一个技能
- 去 `Configs/Skills/skills.yaml`
- 数据结构看 `SkillDefinition.cs`
- 运行逻辑看 `SkillSystem.cs`

3. 新增一个点击事件 / 挂机事件
- 去 `Configs/Events/*.yaml`
- 事件结构看 `EventDefinition.cs`
- 运行逻辑看 `ClickEventSystem.cs` / `IdleSystem.cs`

4. 改背包、金币、库存
- 去 `Scripts/Core/Runtime/`
- 背包看 `PlayerInventory.cs`
- 金币看 `PlayerEconomyState.cs`
- 商店库存看 `PlayerShopState.cs`

5. 改启动流程
- 去 `project.godot`
- 再看 `Scripts/Autoload/GameManager.cs`
- 再看 `Scenes/Main.tscn`


================================
四、当前结构最重要的分层原则
================================

请尽量记住这 4 句话：

1. `Configs/` 是静态内容，不是玩家存档
2. `Definitions` 是模板，`Runtime` 是玩家当前状态
3. `Registry` 负责查数据，`Systems` 负责改数据
4. `Scenes` / `UI` 负责显示，尽量不要在 UI 里直接硬写业务规则
