# 42_子场景标题铭牌样式调整说明_0413

## 1. 本次为什么要改

当前中央区域里：

- 子场景标题
- 子场景下方的事件按钮

两者之前都接近“圆角长条底色 + 上方文字”的结构，所以视觉上容易混在一起。

用户指出的问题是对的：

- 商店名和商店下方事件的区分度不足
- 子场景标题条高度偏高
- 子场景标题条看起来仍然太像按钮

因此这次的目标不是让标题更花，而是让它更像“栏目标题”，不再像“可点击事件按钮”。

---

## 2. 本次采用的方案

本次没有采用：

- 闪烁文字
- 多彩渐变文字

因为这两种方案虽然显眼，但更像特殊事件或稀有按钮，不像稳定的场景标题。

本次最终采用的是：

- 低矮铭牌式标题条

具体做法：

1. 保留用户认可的浅灰半透明底色  
   继续使用接近 `30%` 透明白色的背景。

2. 降低标题条高度  
   通过压缩上下内边距，让标题条只比文字略高。

3. 改掉按钮感  
   不再使用明显的大圆角长条按钮观感，而是改成：
   - 小圆角
   - 上下细边
   - 更扁的铭牌结构

4. 增加顶部细装饰线  
   在标题条最上方加一条 2 像素细线，让它更像“栏目铭牌”，而不是“操作按钮”。

5. 标题字号略微下调  
   让标题更紧凑，也进一步减少高度占用。

---

## 3. 本次改动文件

- `Scripts/UI/GamePlayUI.Layout.cs`
- `Scripts/UI/GamePlayUI.Actions.cs`

其中：

- `GamePlayUI.Layout.cs`
  负责子场景标题条的实际结构
- `GamePlayUI.Actions.cs`
  新增了专门的 `CreateSceneHeaderStyle()` 样式方法

---

## 4. 当前子场景标题条的结构

当前结构已经变成：

1. 外层标题铭牌容器
2. 顶部细装饰线
3. 低高度文字区

而不是原先更接近按钮的：

1. 大圆角长条
2. 中间文字

所以现在它和下方事件按钮的层级关系会更清晰：

- 标题是“区块头”
- 按钮是“操作项”

---

## 5. 当前保留的规则

后续如果继续调 UI，建议保留以下规则：

1. 子场景标题不要做闪烁
2. 子场景标题不要做高饱和彩虹字
3. 子场景标题条高度应明显低于按钮
4. 子场景标题条应保持“铭牌感”，不是“按钮感”

只要这四条还在，标题和事件按钮就不容易再混在一起。
