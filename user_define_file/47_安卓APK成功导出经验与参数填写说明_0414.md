## 2. 这次真正跑通的关键改动

### 2.1 Android 导出使用 .NET 9，日常开发继续使用 .NET 8

这部分是本次成功的基础前提。

已经处理到位的文件：

- [test_00_0410.csproj](/l:/00_godot_project_0410/test-00-0410/test_00_0410.csproj)
- [Directory.Build.props](/l:/00_godot_project_0410/test-00-0410/Directory.Build.props)

当前策略是：

- Windows / 编辑器日常开发：`net8.0`
- Android 导出：`net9.0`

也就是说：

- 你本机新装的 `.NET 9` 不会把整个项目开发链强行变成 `.NET 9`
- 平时继续跑编辑器和 Windows 版时，仍然按 `.NET 8` 逻辑工作
- 只有在 `GodotTargetPlatform=android` 或 `RuntimeIdentifier=android-*` 时，才切到 `net9.0`

这是为了兼容 Godot 4.6 Mono 当前的 Android C# 导出要求。

这样可以避免导出资源时把临时构建目录、缓存目录、Gradle 工作目录反复打进 APK。

### 2.4 本次真正导致“Unable to setup the Godot engine”的错误点

根因在这里：

- [GodotApp.java](/l:/00_godot_project_0410/test-00-0410/android/build/src/main/java/com/godot/game/GodotApp.java)

之前这里被额外插入过一段自定义启动参数逻辑，会强行给 Godot 注入：

```text

```

这会破坏 Godot Android Mono 的正常启动装配流程，结果就是：

- APK 可以安装
- 资源看起来也在
- Mono 原生 `.so` 也可能在
- 但启动时 Godot 引擎初始化失败，直接弹错误框并退出

本次已经把这段自定义 `getCommandLine()` 覆盖逻辑移除，恢复为 Godot 默认启动方式。

这是这次实机最终能跑起来的决定性修复。

## 3. 这次成功导出的稳定流程

下次如果要继续打安卓测试包，优先复用下面这套流程。

### 3.1 先确认环境

至少确认这些项目：

- Godot：`4.6.2 Mono`
- Android Export Templates：`4.6.2.stable.mono`
- JDK：`21`
- Android SDK：可用
- build-tools：`35.0.1`
- compileSdk / targetSdk：`35`
- NDK：`28.1.x`
- Android 导出链可用 `.NET 9`

### 3.2 不要只跑 Gradle 半流程

这次已经验证过：

- 单独跑 `assembleMonoDebug`，有时会得到一个 **能安装但不完整** 的 APK
- 之前 39MB 的包就是这种情况，里面缺少 Mono/.NET 原生运行库

因此稳定做法是：

1. 先保证 Godot 自己完成 Android 导出阶段
2. 让 Godot 负责拷贝 Android Mono 所需原生库和托管发布文件
3. 再由 Gradle 收尾打成最终 APK

最稳的方式就是直接使用 Godot 的完整导出命令。

### 3.3 这次成功使用的完整导出命令

```powershell
$env:JAVA_HOME='C:\Program Files\Microsoft\jdk-21.0.1.12-hotspot'
$env:ANDROID_HOME='L:\Android\Sdk'
$env:ANDROID_SDK_ROOT='L:\Android\Sdk'
$env:GRADLE_USER_HOME='L:\00_godot_project_0410\test-00-0410\.gradle-user-home'
$env:DOTNET_CLI_HOME='L:\00_godot_project_0410\test-00-0410\.dotnet_cli'
$env:NUGET_PACKAGES='L:\00_godot_project_0410\test-00-0410\.dotnet_cli\.nuget\packages'
$env:DOTNET_MULTILEVEL_LOOKUP='0'
$env:DOTNET_SKIP_FIRST_TIME_EXPERIENCE='1'
$env:Path='C:\Program Files\Microsoft\jdk-21.0.1.12-hotspot\bin;' + $env:Path

& 'L:\new_workspace_0410\Godot_v4.6.2-stable_mono_win64\Godot_v4.6.2-stable_mono_win64\Godot_v4.6.2-stable_mono_win64.exe' `
  --headless `
  --path 'L:\00_godot_project_0410\test-00-0410' `

  --verbose `
  --log-file 'L:\00_godot_project_0410\test-00-0410\android_export_debug_0414.log'
```

### 3.4 导出完成后要检查什么

至少检查这几个点：

1. APK 体积不能明显异常偏小。
   这次可运行包约为 `107 MB`。
2. APK 内应包含这些原生库：
   - `libmonosgen-2.0.so`
   - `libmono-component-*.so`

3. APK 内应包含：

4. 启动入口里不能再有自定义的 `assets://android_data_test.pck` 注入逻辑。

## 4. 这次踩过的坑

## 4.1 安装成功，不代表运行成功

本次已经遇到两种不同的“假成功”：

1. APK 可以安装，但启动时报 `Unable to setup the Godot engine`
2. APK 大小看起来正常，但仍然因为错误启动参数导致启动失败

所以判断是否成功，必须以 **实机启动通过** 为准，不能只看：

- 是否打包成功
- 是否能安装
- 是否大小看起来像真的

## 4.2 39MB 的 APK 是不完整包

之前生成的约 `39 MB` APK 缺少 Mono/.NET 原生库。

这类包通常来自：

- 只跑了 Gradle
- 或者 Godot 没有先把 Mono Android 所需文件复制到 `android/build` 工作目录

这种 APK 有时能装上，但并不是真正可运行的 Mono 安卓包。

## 4.3 102MB 的 APK 也可能仍然失败

你手动打出来的约 `102 MB` APK 虽然已经包含 Mono 原生库和托管 DLL，但它当时仍然会报同样的引擎初始化错误。

这说明：

- “是否带齐 Mono 库”只是必要条件
- 不是充分条件

真正的根因最终还是 Android 启动入口那段多余的 `--main-pack` 覆盖。

## 4.4 Codex/受限环境里，Godot 导出可能因为日志目录权限导致假失败

这次还遇到过一个很隐蔽的问题：

- Godot 在内部调用 MSBuild 时，会写 `%AppData%\Godot\mono\build_logs\...`
- 如果当前工具会话对那个目录写权限受限，就可能出现：

```text
MSBUILD : logger error MSB1029
Failed to create log file
Access to the path ... is denied
```

这类问题看起来像“导出失败”，但本质上是日志写入权限导致。

如果以后还用 Codex 继续打包安卓，记得：

- Godot 完整导出阶段往往需要放开到正常系统目录的写权限

## 4.5 Java / Gradle 常驻不等于还在构建

这次还踩到一个观察误区：

- 构建完成后，Gradle daemon 和 Java 进程可能继续常驻
- 不能单看进程还在，就判断为“还没打完”

更准确的判断方式是看：

- APK 输出文件时间戳
- APK 大小是否更新
- Godot 导出日志是否已经走到 Gradle 阶段

## 6. 专门写给开发者的说明：如果你想进一步完善安卓端，需要补哪些内容

下面这些内容，你都 **不需要去 Android Studio 里手填**，后续完全可以交给我按 YAML 配。

### 6.1 包名怎么定

包名就是 Android 的唯一应用 ID。

推荐格式：

```text
com.你的标识.你的游戏名
```

规则：

- 全小写
- 只用字母、数字、点
- 不能有空格
- 每段不要用中文

例如：

```text
com.yourstudio.idletextgame
```

注意事项：

- 包名一旦改了，Android 会把它当成一个新应用
- 同一台手机上，旧包名的版本不能直接覆盖成新包名

### 6.2 应用名怎么定

应用名就是安装后桌面上显示的名字。

建议：

- 控制在 4 到 12 个汉字，或 8 到 20 个英文字符以内
- 不要太长，否则不同机型上可能被截断

例如：

- `文字挂机测试版`
- `Idle Text Test`

### 6.3 图标应该怎么准备

安卓图标这里最容易出问题，建议按下面标准准备。

#### 主图标

- 文件格式：`PNG`
- 文件路径：由 YAML 里填写
- 推荐尺寸：`192x192`
- 文件背景：可以透明
- 不要用 `.ico`

#### 自适应图标

Android 8.0 及以上建议再准备三张：

1. 前景图
2. 背景图
3. 单色图

建议规格：

- 文件格式：`PNG`
- 尺寸：`432x432`

建议做法：

- 前景图：主体图形，建议透明底
- 背景图：纯色或简单渐变底
- 单色图：纯单色版本，方便系统主题适配

额外建议：

- 图形主体不要贴边，四周至少留 10% 到 15% 安全边距
- 尽量避免很细的线条，不然在小图标上会糊

### 6.4 图标路径以后怎么填

以后你只需要在 YAML 里填这种项目内路径即可：

```text
user_define_file/android_icons/replace_me_main_192x192.png
```

也就是说：

- 图标你放进项目里
- 我读取 YAML
- 我把图标路径写入 Godot 安卓导出参数

你不需要自己打开 Android Studio 去配 mipmap。

### 6.5 签名怎么处理

测试阶段：

- 继续用 debug keystore 就够了

正式发布阶段：

- 必须换成你自己的 release keystore

非常重要：

- release keystore 一定要妥善备份
- 一旦丢失，后续正式包升级会非常麻烦

你以后只需要在 YAML 填：

- keystore 文件路径
- alias
- password

我就可以按这些参数导出。

### 6.6 权限怎么处理

建议原则是：

- 没需求就不要开
- 测试版尽量最小权限

目前这个项目如果只是本地单机测试，一般常用的是：

- `INTERNET`：如果以后有联网、热更、远程配置
- `ACCESS_NETWORK_STATE`：如果要判断网络状态
- `POST_NOTIFICATIONS`：如果以后要做通知
- `WAKE_LOCK`：如果以后有挂机保活相关需求
- `VIBRATE`：如果要做震动反馈

如果当前只是本地测试剧情和存档，完全可以先全部关闭。

### 6.7 版本号要怎么管理

安卓里至少有两个版本字段：

- `version_name`
- `version_code`

建议：

- `version_name` 给人看，例如：`0.1.0`
- `version_code` 给系统识别，必须递增，例如：`1`、`2`、`3`

注意：

- 如果以后装机升级失败，很常见的原因就是 `version_code` 没递增

### 6.8 ABI 架构怎么选

当前测试阶段建议：

- 只打 `arm64-v8a`

原因：

- 现在大多数安卓手机都支持
- 包会更小
- 问题更少

如果以后要兼容更多老设备，再考虑加：

- `armeabi-v7a`

### 6.9 屏幕方向与分辨率适配

当前项目现在按横屏测试更稳。

如果以后你想改：

- 横屏
- 竖屏
- 自动旋转

也建议统一写进 YAML，由我同步进导出参数。

### 6.10 正式版前还应该补哪些工作

如果以后不是只做测试包，而是准备更正式的安卓版本，建议补上这些项：

1. 正式应用图标
2. 正式应用名
3. 正式包名
4. 正式 release 签名
5. 版本号策略
6. 最小权限清单
7. 横竖屏策略
8. 首次启动与存档权限验证
9. 多机型实机测试
10. 一次完整的 ADB 日志留档

## 7. YAML 字段与导出项的对应关系

以后你只需要优先维护 YAML，不需要自己去 Android Studio 里配置。

常见映射关系如下：

- `package_name` -> Godot Android 导出的 `package/unique_name`
- `application_name` -> Godot Android 导出的 `package/name`
- `version_name` -> `version/name`
- `version_code` -> `version/code`
- `min_sdk` -> `gradle_build/min_sdk`
- `target_sdk` -> `gradle_build/target_sdk`
- `icons.main_192x192` -> `launcher_icons/main_192x192`
- `icons.adaptive_foreground_432x432` -> `launcher_icons/adaptive_foreground_432x432`
- `icons.adaptive_background_432x432` -> `launcher_icons/adaptive_background_432x432`
- `icons.adaptive_monochrome_432x432` -> `launcher_icons/adaptive_monochrome_432x432`
- `permissions.*` -> Godot Android 导出里的权限开关
- `signing.*` -> Gradle / Godot 导出签名参数
