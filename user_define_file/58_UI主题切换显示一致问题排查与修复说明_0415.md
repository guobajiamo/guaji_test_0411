# 58_UI主题切换显示一致问题排查与修复说明_0415

## 1. 问题现象
- 游戏内切换“新UI(stitch)”和“旧UI(legacy)”后，除了顶部少量文字颜色变化，整体界面几乎一致。

## 2. 根因定位
- 主题配置路径选择逻辑使用了 `ResourceLoader.Exists(...)` 来判断 `.yaml` 文件是否存在。
- 但 `.yaml` 不是 Godot 的可导入资源类型，`ResourceLoader.Exists` 对这类文件返回不可靠（本项目中等同于未命中）。
- 导致结果：
  - `GameplayUiConfigLoader` 实际总是回退到 `gameplay_ui_theme.yaml`；
  - `UiImageThemeManager` 实际总是回退到 `ui_image_theme.yaml`；
  - 所以新旧主题大部分视觉都相同，只剩少量代码里按模式硬编码分支的颜色会变化（例如顶部文字）。

## 3. 修复内容

### 3.1 UI布局主题文件存在性判断
- 文件：`Scripts/UI/GameplayUiConfig.cs`
- 修改：将 `ResolveThemeConfigPath(...)` 中的判断从
  - `ResourceLoader.Exists(...)`
  改为
  - `FileAccess.FileExists(...)`
- 目的：确保 `gameplay_ui_theme_legacy.yaml` / `gameplay_ui_theme_stitch.yaml` 能被真实选中。

### 3.2 UI图片主题文件存在性判断
- 文件：`Scripts/UI/UiImageThemeManager.cs`
- 修改：将 `ResolveConfigPath()` 中的判断从
  - `ResourceLoader.Exists(...)`
  改为
  - `FileAccess.FileExists(...)`
- 目的：确保 `ui_image_theme_legacy.yaml` / `ui_image_theme_stitch.yaml` 能被真实选中。

## 4. 验证结果
- `dotnet build .\\test_00_0410.csproj`：通过（0 error / 0 warning）。
- `UiFeatureSmokeTest.tscn` headless 冒烟：通过，日志未见脚本报错。

## 5. 修复后预期
- 切换系统页“新UI/旧UI”按钮后：
  - 主背景、主要面板、标签、状态色等会按各自yaml显著变化；
  - 不再是“仅顶部文字有差异”。

## 6. 后续建议
- 如果你希望新旧差异更强，可在两套yaml里继续拉大：
  - `panels.*.color / border_color`
  - `tabs.*`
  - `status.*`
- 现有切换机制可直接复用到后续“白天/夜晚”主题。
