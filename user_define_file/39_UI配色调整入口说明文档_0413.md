# 39_UI配色调整入口说明文档_0413

## 1. 本次配色调整结论

根据你的要求，这一版已经把 UI 的整体配色从“灰蒙蒙、杂色偏多”的方向，切换成了：

- 整体暗色底
- 更纯净的冷色深背景
- 明显的深蓝 / 深红强调色
- 尽量避免大面积灰绿、土黄、灰棕混用

当前风格可以概括为：

- 背景和大面板：深冷蓝黑
- 主强调色：深蓝
- 次强调色：深红
- 文本：偏冷白

---

## 2. 如果你以后想调 UI 配色，优先改哪两个文件

### 2.1 第一优先级：整体主界面主题

文件：

- `Configs/UI/gameplay_ui_theme.yaml`

这个文件控制的是：

- 主界面背景色
- 顶栏 / 左栏 / 中栏 / 右栏 / 日志栏 / 场景卡片的底色和边框色
- 标签页颜色
- 状态栏文字和进度条颜色
- 区域标题强调色

也就是说：

- 如果你想改“整个游戏内主界面的整体配色气质”，优先改这个文件

---

### 2.2 第二优先级：按钮颜色

文件：

- `Configs/UI/ui_image_theme.yaml`

这个文件控制的是：

- 主菜单按钮颜色
- 各类事件按钮颜色
- 系统按钮颜色
- 物品按钮 / 通用按钮颜色

也就是说：

- 如果你想改“按钮看起来是什么颜色”，优先改这个文件

---

## 3. `gameplay_ui_theme.yaml` 里最重要的配色参数

### 3.1 改整体背景色

位置：

```yaml
general:
  background_color: "#060912"
```

作用：

- 控制整个主界面的最底层背景色

建议：

- 想更偏纯黑，就继续往 `#03060d` 这类方向压
- 想更偏蓝黑，就用 `#07101a`、`#08111d`

---

### 3.2 改各大面板颜色

位置：

```yaml
panels:
  top_bar:
    color: "#0d1424"
    border_color: "#2f6fbe"

  left_column:
    color: "#0b1220"
    border_color: "#326dbe"

  center_column:
    color: "#110f1c"
    border_color: "#8e2447"

  bottom_log:
    color: "#0b1320"
    border_color: "#2a5fa8"

  right_info:
    color: "#0d1422"
    border_color: "#3d79c9"

  right_status:
    color: "#170d18"
    border_color: "#9c2b53"

  scene_card:
    color: "#0d1523"
    border_color: "#315b9f"
```

作用：

- `color` 控制面板底色
- `border_color` 控制边框强调色
- `alpha` 控制面板透明度
- `border_alpha` 控制边框透明度

建议：

- 想让 UI 更统一，就尽量让大部分 `color` 都在同一冷色系里
- 想让重点更明显，就只让少数 `border_color` 用红色或亮蓝
- 不要每个面板都用完全不同的综合色，否则会重新变杂

---

### 3.3 改标签页颜色

位置：

```yaml
tabs:
  normal_color: "#0d1522"
  active_color: "#8a2042"
  normal_text_color: "#dbe7ff"
  active_text_color: "#fff1f5"
  border_color: "#3a72c0"
```

作用：

- 控制中间标签栏未选中 / 选中状态的颜色

建议：

- `normal_color` 用深蓝黑
- `active_color` 用深红或深蓝都行
- 如果你想更理性科技感，用蓝色激活
- 如果你想更强烈视觉焦点，用红色激活

---

### 3.4 改状态栏颜色

位置：

```yaml
status:
  header_text_color: "#f4f7ff"
  body_text_color: "#d7e0f2"
  accent_text_color: "#ff5c86"
  progress_fill_color: "#3d79d8"
  progress_background_color: "#16243d"
```

作用：

- 控制右下状态栏标题、正文、强调文字、进度条颜色

建议：

- 强调字建议和按钮强调色呼应
- 进度条填充色建议和主强调蓝一致

---

### 3.5 改左侧区域标题色

位置：

```yaml
region_accents:
  primary_story:
    text_color: "#61a8ff"
  primary_faction:
    text_color: "#ff668e"
  secondary_peace:
    text_color: "#86bcff"
  secondary_battle:
    text_color: "#ff7a9b"
```

作用：

- 控制左侧区域树里不同类型区域的标题色

建议：

- 一级标题颜色差异可以大一点
- 二级标题颜色建议是一级标题的弱化版

## 4. `ui_image_theme.yaml` 里最重要的配色参数

这个文件主要是按钮颜色。

每一组按钮样式都长这样：

```yaml
event_click:
  fallback_normal_color: "#183f7b"
  fallback_hover_color: "#2155a2"
  fallback_pressed_color: "#122d58"
  fallback_disabled_color: "#55657f"
  text_color: "#edf5ff"
  disabled_text_color: "#cad3e3"
  border_color: "#0f1f3f"
```

### 常用字段说明

- `fallback_normal_color`
  按钮默认颜色
- `fallback_hover_color`
  鼠标移上去时的颜色
- `fallback_pressed_color`
  按下时的颜色
- `fallback_disabled_color`
  禁用态颜色
- `text_color`
  正常文字颜色
- `disabled_text_color`
  禁用文字颜色
- `border_color`
  边框颜色

---

## 5. 按钮样式对应关系

你以后如果只想改某一类按钮，可以按下面找：

- `menu_primary`
  主菜单主按钮
- `menu_test_entry`
  主菜单测试入口按钮
- `event_oneshot`
  一次性事件按钮
- `event_click`
  普通点击事件按钮
- `event_idle`
  挂机事件按钮
- `system_action`
  系统页普通按钮
- `system_return_menu`
  系统页返回主菜单按钮
- `special_action`
  通用特殊按钮，目前也会用于部分物品和技能页按钮

## 6. 如果你想快速换成别的配色，最少改哪些参数

### 方案 A：暗蓝科技感

优先改：

- `gameplay_ui_theme.yaml`
  - `background_color`
  - `panels.*.color`
  - `panels.*.border_color`
  - `tabs.normal_color`
  - `tabs.active_color`
  - `status.progress_fill_color`
- `ui_image_theme.yaml`
  - `menu_primary`
  - `event_click`
  - `system_action`

---

### 方案 B：暗红权谋感

优先改：

- `gameplay_ui_theme.yaml`
  - `center_column.border_color`
  - `right_status.border_color`
  - `tabs.active_color`
  - `status.accent_text_color`
- `ui_image_theme.yaml`
  - `event_oneshot`
  - `event_idle`
  - `system_return_menu`

---

### 方案 C：蓝红双主色

这也是当前这一版的思路：

- 蓝色负责主结构、信息层、系统层
- 红色负责激活、危险、重点和操作焦点

对应主要参数：

- 蓝色：
  - `panels.top_bar.border_color`
  - `panels.left_column.border_color`
  - `panels.right_info.border_color`
  - `status.progress_fill_color`
  - `buttons.event_click.*`
- 红色：
  - `panels.center_column.border_color`
  - `panels.right_status.border_color`
  - `tabs.active_color`
  - `status.accent_text_color`
  - `buttons.event_oneshot.*`
  - `buttons.event_idle.*`

## 7. 哪些颜色不是优先从 YAML 改，而是在代码里写死的

虽然大部分主题已经可从 YAML 改，但当前仍有少量颜色是在代码中直接写的。

如果你以后想进一步完全自定义，可以再看这些文件：

- `Scripts/UI/GamePlayUI.cs`

  - 左栏标题色
  - 当前区域标题色
- `Scripts/UI/GamePlayUI.Layout.cs`
  - 场景标题色
  - 当前区域选中底色
  - 区域计数色
- `Scripts/UI/CharacterStatusPanel.cs`
  - 折叠头按钮底色和边框色
  - 状态卡片边框色
- `Scripts/UI/ConfigurableInfoPanel.cs`
  - 右上信息面板标题与正文颜色
- `Scripts/UI/LogPanel.cs`

也就是说：

- 现在“80% 以上的换色”已经可以只靠两个 YAML 文件完成
- 剩下少量细节色，如果你以后还想完全开放成可配置项，可以再继续外提到 YAML

---

## 8. 最后建议

如果你以后自己继续调色，我建议你遵守这 3 条：

1. 大底色统一成同一冷暗色系，不要左一块右一块不同综合色
2. 真正高饱和的颜色只留给少数强调位，比如按钮激活、边框、重点文字
3. 同时只保留 1 到 2 个主强调色，最稳妥的就是“蓝 + 红”

这样不容易再回到“单个颜色还行，但放一起显得脏和灰”的问题。
