# 41_新UI贴边压缩调整说明文档_0413

## 1. 本轮调整目标

这一轮主要解决两个问题：

1. 底部运行日志在“最小化”状态下没有稳定显示出 1 行文本，看起来像直接被关闭了
2. 当前整套 UI 在 1920x1080 下仍然显得偏松散，没有尽量贴满屏幕边缘

所以这次调整的重点不是换色，而是：

- 修正最小化日志的显示高度
- 压缩整体布局
- 让 UI 更贴边、更紧凑
- 同时保持上一轮已经确认下来的 UI 结构和功能逻辑不变

---

## 2. 日志最小化修正

调整文件：

- `Scripts/UI/GamePlayUI.cs`
- `Scripts/UI/LogPanel.cs`

### 2.1 问题

上一轮虽然已经把底部 LOG 改成：

- 最小化
- 默认 5 行
- 最大化

三段式逻辑，但“最小化”的高度仍然偏低，导致视觉上几乎看不到日志正文，等于接近被关闭。

### 2.2 本次修正

现在把最小化状态的 LOG 高度进一步抬高，确保：

- 头部按钮栏仍可见
- 文本区至少能稳定显示 1 行日志内容

同时 `LogPanel` 仍然保留：

- 默认折叠显示最近 5 行
- 最小化折叠显示最近 1 行
- 最大化显示更多完整日志

### 2.3 当前底部 LOG 的最终状态

当前逻辑如下：

1. 默认状态  
   显示约 5 行日志

2. 最小化状态  
   点击“最小化”后，缩为单行日志显示

3. 最大化状态  
   点击“最大化”后，弹出悬浮窗显示更多日志

4. 从最大化状态点“收起”  
   关闭悬浮窗，并回到默认 5 行高度

## 3. 整体布局压缩与贴边

调整文件：

- `Configs/UI/gameplay_ui_theme.yaml`
- `Scripts/UI/GamePlayUI.Actions.cs`
- `Scripts/UI/GamePlayUI.cs`

### 3.1 外边距缩小

本次把全局 `outer_margin` 大幅缩小。

效果：

- 顶栏更靠上
- 左栏更靠左
- 右栏更靠右
- 底部日志更靠下

也就是说，整个 UI 会更接近“占满 1080P 界面”的感觉，而不是四周留出明显空白。

### 3.2 区块间距缩小

本次继续缩小：

- `section_gap`
- `inner_gap`

效果：

- 左中右三大列之间更紧
- 中间内容与日志之间更紧
- 子场景卡片之间也比之前更收

### 3.3 边框再加粗一档

为了减少“框体之间像被缝隙切开”的感觉，这次又把主面板边框和标签边框再加粗了一档。

这样做的目的不是单纯让线变重，而是：

- 让边框更有结构感
- 在视觉上吃掉一部分缝隙
- 让整套 UI 更像一体化面板，而不是一堆散开的块

---

## 4. 中间标签内容区继续放大

调整文件：

- `Configs/UI/gameplay_ui_theme.yaml`
- `Scripts/UI/GamePlayUI.cs`

当前通过以下方式让中间内容区进一步变大：

- 左右两栏宽度略缩
- 外边距明显缩小
- 区块间隙缩小

因此在 1920x1080 下，中间标签页的实际可用面积会比之前更大。

---

## 5. 子场景区继续保持的规则

虽然这轮主要是布局压缩，但仍保留了上一轮已经定下来的子场景规则：

- 子场景标题条使用浅灰半透明色块
- 标题条高度保持较低
- 不再使用之前那种偏彩色、偏厚重的标题条

这部分仍然是当前“子场景区”需要保持的基础风格，不建议后续轻易改回去。
